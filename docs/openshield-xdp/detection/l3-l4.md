# L3/L4 Detection

OpenShield validates network and transport layer headers before any rate tracking. These checks are **zero-tolerance** — any violation results in an immediate `XDP_DROP`.

## L3: IP validation

### IPv4 bogon filter

Seven bogon/private ranges are checked in `validate_source_ip`:

```c
// All comparisons on host byte order (bpf_ntohl applied once)
if ((ip & 0xFF000000) == 0x0A000000)    → DROP  // 10.0.0.0/8
if ((ip & 0xFFF00000) == 0xAC100000)    → DROP  // 172.16.0.0/12
if ((ip & 0xFFFF0000) == 0xC0A80000)    → DROP  // 192.168.0.0/16
if ((ip & 0xFF000000) == 0x7F000000)    → DROP  // 127.0.0.0/8
if ((ip & 0xFFFF0000) == 0xA9FE0000)    → DROP  // 169.254.0.0/16
if ((ip & 0xFF000000) == 0x00000000)    → DROP  // 0.0.0.0/8
if ((ip & 0xF0000000) == 0xE0000000)    → DROP  // 224.0.0.0/4
```

| Range | CIDR | Description | Drop reason |
|-------|------|-------------|-------------|
| `10.0.0.0/8` | 10.0.0.0/8 | Private (RFC 1918) | `DROP_PRIVATE_SOURCE` |
| `172.16.0.0/12` | 172.16.0.0/12 | Private (RFC 1918) | `DROP_PRIVATE_SOURCE` |
| `192.168.0.0/16` | 192.168.0.0/16 | Private (RFC 1918) | `DROP_PRIVATE_SOURCE` |
| `127.0.0.0/8` | 127.0.0.0/8 | Loopback | `DROP_PRIVATE_SOURCE` |
| `169.254.0.0/16` | 169.254.0.0/16 | Link-local (RFC 3927) | `DROP_PRIVATE_SOURCE` |
| `0.0.0.0/8` | 0.0.0.0/8 | "This" network (RFC 1122) | `DROP_PRIVATE_SOURCE` |
| `224.0.0.0/4` | 224.0.0.0/4 | Multicast | `DROP_PRIVATE_SOURCE` |

### IPv6 bogon filter

Six bogon/private ranges are checked in `validate_source_ip6`:

```c
// ::1/128 — loopback
if (hi == 0 && lo == 0x0100000000000000ULL)  → DROP

// ::/128 — unspecified
if (hi == 0 && lo == 0)                       → DROP

// fe80::/10 — link-local
if (fb == 0xfe && ((hi >> 8) & 0xC0) == 0x80) → DROP

// fc00::/7 — unique local
if ((fb & 0xFE) == 0xFC)                      → DROP

// ff00::/8 — multicast (invalid as source)
if (fb == 0xFF)                               → DROP

// ::ffff:0:0/96 — IPv4-mapped
if (hi == 0 && ((lo >> 32) & 0xFFFF) == 0xFFFF) → DROP
```

| Range | CIDR | Description |
|-------|------|-------------|
| `::1` | ::1/128 | Loopback |
| `::` | ::/128 | Unspecified address |
| `fe80::/10` | fe80::/10 | Link-local |
| `fc00::/7` | fc00::/7 | Unique local (ULA) |
| `ff00::/8` | ff00::/8 | Multicast (invalid as source) |
| `::ffff:0:0/96` | ::ffff:0:0/96 | IPv4-mapped addresses |

::: info IPv6 byte order
IPv6 addresses are stored as two `u64` fields (`hi`, `lo`). The first address byte is the **lowest byte** of `hi` on little-endian hosts. Validation uses bitwise operations on the raw u64 values, not byte-level comparisons — this works correctly on both little-endian and big-endian architectures.
:::

## L4: TCP flag validation

Five bogus TCP flag combinations are always dropped:

```c
// SYN+FIN — mutually exclusive (open + close simultaneously)
if ((tcp_flags & TCP_SYN) && (tcp_flags & TCP_FIN))   → DROP_BOGUS_TCP

// SYN+RST — mutually exclusive
if ((tcp_flags & TCP_SYN) && (tcp_flags & TCP_RST))   → DROP_BOGUS_TCP

// FIN+RST — mutually exclusive (close + reset)
if ((tcp_flags & TCP_FIN) && (tcp_flags & TCP_RST))   → DROP_BOGUS_TCP

// All 8 flags set — impossible, used in Xmas scans
if ((tcp_flags & TCP_VALID_MASK) == TCP_VALID_MASK)    → DROP_BOGUS_TCP

// No flags set — NULL scan
if ((tcp_flags & TCP_VALID_MASK) == 0)                 → DROP_BOGUS_TCP
```

| Combination | Attack vector | Flag bits |
|------------|--------------|-----------|
| SYN+FIN | Protocol fuzzing / OS fingerprinting | `0x03` |
| SYN+RST | TCP reset attacks | `0x06` |
| FIN+RST | Protocol violation | `0x05` |
| All flags | Xmas scan / Nmap | `0xFF` |
| No flags | NULL scan / Nmap | `0x00` |

::: tip TCP flags are raw
The `tcp_flags` field is the raw 13th byte of the TCP header, containing all 8 flag bits. CWR and ECE flags are included in the "all flags" check via `TCP_VALID_MASK`. Legitimate combinations (SYN, SYN+ACK, ACK, FIN, RST, FIN+ACK, etc.) pass through.
:::

### Valid flag combinations (allowed through)

| Flags | Description | Hex |
|-------|-------------|-----|
| SYN | Connection request | `0x02` |
| SYN+ACK | Connection acknowledgment | `0x12` |
| ACK | Data acknowledgment | `0x10` |
| FIN | Connection close | `0x01` |
| RST | Connection reset | `0x04` |
| FIN+ACK | Close acknowledgment | `0x11` |
| RST+ACK | Reset acknowledgment | `0x14` |
| PSH+ACK | Data push | `0x18` |
| URG+ACK | Urgent data | `0x30` |

## L4: Header bounds validation

After TCP flag validation, the L4 header is checked for minimum bounds:

```c
if (proto == TCP) {
    // Must have at least 20 bytes (struct tcphdr)
    if ((void *)(tcp + 1) > data_end) → DROP_MALFORMED_L4
}
if (proto == UDP) {
    // Must have at least 8 bytes (struct udphdr)
    if ((void *)(udp + 1) > data_end) → DROP_MALFORMED_L4
}
if (proto == ICMP || proto == ICMPv6) {
    // Must have at least 8 bytes (struct icmphdr)
    if ((void *)(icmp + 1) > data_end) → DROP_MALFORMED_L4
}
```

The L4 offset is determined during packet parsing (stage 1) and accounts for variable-length IPv4 headers (ihl × 4) and IPv6 extension headers (up to 4 extension headers totaling up to 256 bytes + 40 fixed = 296 max offset).

## VLAN support

The packet parser handles up to 2 nested VLAN tags:

```c
// Handle up to 2 VLAN tags (802.1Q / 802.1ad / QinQ)
#pragma unroll
for (int i = 0; i < 2; i++) {
    if (ethertype != bpf_htons(0x8100) && ethertype != bpf_htons(0x88A8))
        break;
    ethertype = *(u16 *)(data + l2_start + 2);
    l2_start += 4;  // Each tag adds 4 bytes
}
```

| Protocol | EtherType | Description |
|----------|-----------|-------------|
| 802.1Q | `0x8100` | Standard VLAN tagging |
| 802.1ad | `0x88A8` | Provider Bridging (QinQ outer tag) |

With 2 tags (QinQ), the total L2 header is `14 + 4 + 4 = 22` bytes. The `l2_offset` field tracks this dynamically.

## Configuration

```yaml
validation:
  filter_private: true           # IPv4/IPv6 private/bogon source filtering
  filter_bogon: true             # Legacy alias (synonym for filter_private)
  filter_bogus_tcp: true         # Bogus TCP flag detection
  filter_malformed: true         # L4 header bounds validation
```

All validation flags can be disabled independently. Whitelisted IPs with `WL_SKIP_VALIDATION` bypass L3 and L4 validation but NOT UDP amplification or L7 signature checks.
